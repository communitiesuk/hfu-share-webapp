    This accessibility statement applies to the Share Homes for Ukraine data (Share) system.

    This website is run by the Ministry for Housing, Communities and Local Government (MHCLG). We want as many people as possible to be able to use this website. For example, that means you should be able to:

    • change colours, contrast levels and fonts using browser or device settings
    • navigate most of the website using a keyboard or speech recognition software
    • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

    We've also made the website text as simple as possible to understand.

    AbilityNet has advice on making your device easier to use if you have a disability.

    How accessible this website is

    We know some parts of this website are not fully accessible:

    • some descriptive links are missing
    • screen magnification on some components does not work without the text covering other text
    • some of the link text is not in the usual link text colour
    • some speech recognition and keyboard functionality does not work fully on the service

    Feedback and contact information

    If you find any problems not listed on this page or think we’re not meeting accessibility requirements, raise a support ticket.

    If you need information on this website in a different format like accessible PDF, large print, easy read, audio recording or braille:

    • Raise a support ticket
    • call 0303 4444 445 Monday to Friday 9am to 6pm

    We'll consider your request and get back to you within 5 working days.

    Enforcement procedure

    The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

    Technical information about this website’s accessibility

    MHCLG is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

    Compliance status

    The website has been tested against the Web Content Accessibility Guidelines (WCAG) 2.2 AA standard.

    This website is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to the non-compliances listed below.

    Non-accessible content

    The content listed below is non-accessible for the following reasons.

    Non-compliance with the accessibility regulations

    On some screens text in our top navigation bar cannot be resized without assistive technology, so if people magnify it up to and over 200% the text sits on top of other text. This fails WCAG success criterion 1.4.4 Resize Text (Level AA).

    Descriptive links are missing for most of the buttons on the service. This fails WCAG 2.2 success criterion:

    • 2.4.4 (AA) Link Purpose (In Context)
    • 2.4.9 (AAA) Link Purpose (Link only)
    • 1.3.1 (A) Info and relationships
    • 4.1.2 (A) Name, role, value

    When using a screen reader some of the pronunciations of acronyms like UTLA and LTLA are pronounced incorrectly. This fails WCAG 2.2 success criterion:

    • 3.1.4 (AAA) Abbreviations – a mechanism is available for identifying the expanded form or meaning of abbreviations
    • 3.1.2 (AA) Language of parts
    • 3.1.1 (A) Language of page
    • 1.3.1 (A) Info and relationships

    When requesting access to the service and selecting your relevant Local Authority (LA) a screen reader will not read out the LA. This fails WCAG 2.2 success criterion:

    • 1.3.1 (A) Info and relationships
    • 4.1.2 (A) Name, role, value
    • 1.3.2 (A) Meaningful sequence

    When requesting access to the service if you need to change the LA you have selected, the dropdown box is difficult to select. It may take 5 or 6 clicks. This fails WCAG 2.2 success criterion:

    • 2.1.1 (A) Keyboard accessibility
    • 3.2.1 (A) On focus – when a component receives focus, it must not trigger unexpected context changes

    When requesting access to the service, when you select your LA, the down arrow on the drop-down list is not accessible using a keyboard only. This fails WCAG 2.2 success criterion:

    • 2.1.1 (A) Keyboard accessibility – all functionality must be operable through a keyboard
    • 2.4.7 (AA) Focus visible
    • 1.3.1 (A) Info and relationships

    When requesting access to the service, when you select your LA. You cannot select an option from the dropdown with voice control only. This fails WCAG 2.2 success criterion:

    • 4.1.2 (A) Name, role, value
    • 1.3.1 (A) Info and relationships

    On form pages. The keyboard navigation will skip straight to the text box instead of going through all the content on the page. This fails WCAG 2.2 success criterion:

    • 2.4.3 (A) Focus order
    • 1.3.1 (A) Info and relationships
    • 2.4.6 (AA) Headings and labels
    • 1.3.2 (A) Meaningful sequence

    When pressing Enter on a keyboard to select an option from radio buttons, an error message ‘field is required’ is triggered. This fails WCAG 2.2 success criterion:

    • 2.1.1 (A) Keyboard accessibility
    • 3.3.1 (A) Error identification
    • 3.2.2 (A) On input

    On the home page, you can't access approved, requested or rejected request links using your voice. This fails WCAG 2.2 success criterion:

    • 2.4.3 (A) Focus order
    • 2.4.4 (A) Link purpose (in context)
    • 1.3.1 (A) Info and relationships

    We plan to fix these issues as soon after launch on Monday 15 September, as we can.

    Preparation of this accessibility statement

    This statement was prepared on Monday 15 September 2025. It has not been reviewed yet.

    We plan to test this website in October 2025 against the WCAG 2.2 AA standard.

    The test was carried out by our user research team using assistive technologies available to them and following the WCAG 2.2 criteria. A further audit of the website is due to be carried out to the WCAG 2.2 AA standard in October.
